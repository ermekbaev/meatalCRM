# Инструкция по тестированию MetalCRM

## Подготовка
1. Запустить: `npm run dev`
2. Открыть: http://localhost:3000
3. Войти: `admin@metalcrm.ru` / `admin123`

---

## 1. Авторизация

| Действие | Ожидаемый результат |
|----------|---------------------|
| Открыть `/login` без авторизации | Показывается страница входа |
| Ввести неверный пароль | Ошибка "Неверный email или пароль" |
| Нажать иконку глаза в поле пароля | Пароль показывается/скрывается |
| Вставить пароль с пробелами | Пробелы автоматически удаляются |
| Войти с правильными данными | Редирект на `/dashboard` |
| Открыть `/dashboard` без авторизации | Редирект на `/login` |

---

## 2. Dashboard

| Действие | Ожидаемый результат |
|----------|---------------------|
| Открыть дашборд | Показываются карточки: Заявок, Контрагенты, КП, Выручка |
| Посмотреть график | Отображается пирог по статусам заявок |
| Кликнуть на заявку в "Последние заявки" | Переход на страницу заявки |
| Кликнуть "Все →" | Переход на `/requests` |
| Кликнуть на карточку статуса (Новая/В работе/...) | Переход на список с фильтром по статусу |

---

## 3. Клиенты

| Действие | Ожидаемый результат |
|----------|---------------------|
| Открыть `/clients` | Список клиентов с поиском |
| Нажать "Добавить клиента" | Открывается диалог |
| Создать клиента (только имя) | Клиент появляется в таблице |
| Создать клиента с ИНН, телефоном, email, адресом | Все поля сохраняются |
| Найти клиента через поиск | Список фильтруется по имени/ИНН/телефону |
| Нажать иконку карандаша | Диалог открывается с данными клиента |
| Изменить и сохранить | Данные обновляются в таблице |
| Нажать иконку глаза | Переход на страницу клиента |
| На странице клиента — нажать "Новая заявка" | Переход на форму создания заявки с выбранным клиентом |
| На странице клиента — кликнуть на заявку | Переход на страницу заявки |
| Удалить клиента | Клиент исчезает из списка |

---

## 4. Заявки

| Действие | Ожидаемый результат |
|----------|---------------------|
| Открыть `/requests` | Список заявок с фильтрами |
| Нажать "Новая заявка" | Переход на форму |
| Создать заявку (заполнить название, выбрать клиента) | Создаётся и открывается заявка |
| Выбрать услуги из каталога при создании | Услуги добавляются как теги |
| Фильтровать по статусу | Список обновляется |
| Фильтровать по приоритету | Список обновляется |
| Искать по тексту | Список фильтруется |
| Нажать "Экспорт Excel" | Скачивается .xlsx файл |
| Открыть заявку → изменить статус | Статус меняется мгновенно, появляется в истории |
| Открыть заявку → изменить приоритет | Приоритет меняется |
| Открыть заявку → назначить ответственного | Выбор из списка пользователей, сохраняется |
| Открыть заявку → изменить сумму | Сохраняется при потере фокуса |
| Написать комментарий и нажать кнопку отправки | Комментарий появляется в ленте |
| Написать комментарий и нажать Ctrl+Enter | Комментарий отправляется |
| Посмотреть историю изменений | Все изменения статуса/приоритета/ответственного видны |
| Нажать "Создать КП" | Переход на форму КП со связанной заявкой |

---

## 5. Коммерческие предложения

| Действие | Ожидаемый результат |
|----------|---------------------|
| Открыть `/offers` | Список КП с фильтрами по статусу |
| Нажать "Создать КП" | Переход на форму |
| Выбрать заявку в форме | Подтягивается клиент |
| Добавить позицию вручную | Поля: услуга, описание, кол-во, ед., цена |
| Начать печатать название услуги | Выпадает список из каталога (автодополнение) |
| Выбрать услугу из автодополнения | Цена и единица подставляются автоматически |
| Изменить количество/цену | Сумма пересчитывается |
| Добавить несколько позиций | Итог суммируется |
| Указать скидку % | Итог пересчитывается со скидкой |
| Сохранить КП | Редирект на страницу КП |
| На странице КП — нажать "Скачать PDF" | Загружается PDF с кириллическим текстом (не кракозябры!) |
| Проверить PDF | Название, клиент, таблица позиций, итог — всё читаемо |
| Изменить статус КП | Сохраняется |
| Кликнуть на заявку в КП | Переход на страницу заявки |
| Кликнуть на клиента в КП | Переход на страницу клиента |

---

## 6. Справочник услуг

| Действие | Ожидаемый результат |
|----------|---------------------|
| Открыть `/settings/catalog` | Услуги сгруппированы по категориям, колонки выровнены |
| Нажать "Добавить услугу" | Открывается диалог |
| Кликнуть на поле "Категория" | Выпадает список существующих категорий |
| Выбрать существующую категорию | Услуга попадает в нужную таблицу |
| Написать новую категорию | Создаётся новый раздел |
| Добавить услугу без категории | Появляется в блоке "Без категории" |
| Редактировать услугу | Диалог с заполненными данными |
| Удалить услугу | Исчезает из списка |

---

## 7. Пользователи (только для admin)

| Действие | Ожидаемый результат |
|----------|---------------------|
| Открыть `/settings/users` | Список пользователей |
| Создать пользователя (менеджер) | Появляется в списке |
| Войти под менеджером | Доступ к заявкам, клиентам, КП; в дропдауне "Ответственный" — список пользователей |
| Заблокировать пользователя через тоггл | Пользователь не может войти |
| Войти под заблокированным | Ошибка входа |

---

## Известные ограничения (не баги)
- Загрузка файлов к заявкам — поле в БД есть, UI не реализован
- Отправка КП по email — не реализована
- DaData (автозаполнение по ИНН) — не подключена
